    <div class="block">
      <p><strong>Goal:</strong> Become proficient in Kubernetes, Observability (VictoriaMetrics, OpenTelemetry), Logging, Debugging, and SRE. <strong>Budget:</strong> 150 hours over 20 days = <strong>7.5 focused hours/day</strong>. <strong>Method:</strong> 80/20 prioritization + meta-learning (active recall, spaced repetition, interleaving, deliberate practice). <strong>Platform:</strong> KodeKloud (hands-on labs + playgrounds).</p>
<h2>How to Use This Plan (Meta-Learning Operating System)</h2>
<p>The 80/20 rule drives topic selection: ~20% of concepts (control loops, the 3 pillars, PromQL, SLOs, the debugging method) explain ~80% of real SRE work. Every day is built around <strong>doing</strong>, not reading.</p>
<p><strong>The 5 fast-learning levers used every day:</strong></p>
<ol><li><strong>Active recall</strong> — Close the notes, answer the 50 questions from memory. Retrieval beats re-reading 3:1.</li><li><strong>Spaced repetition</strong> — Re-test yesterday&#x27;s questions each morning; put missed ones into an Anki deck (review daily).</li><li><strong>Interleaving</strong> — Mix labs and topics rather than blocking; it feels harder but sticks better.</li><li><strong>Deliberate practice</strong> — Target the edge of your ability. When a lab feels easy, break it on purpose and fix it.</li><li><strong>Feynman technique</strong> — At day&#x27;s end, explain the day&#x27;s hardest concept out loud in plain language. Gaps = tomorrow&#x27;s review.</li></ol>
<p><strong>Daily learning ritual (baked into every schedule below):</strong></p>
<ul><li><strong>Prime (10 min):</strong> Skim the day&#x27;s topics and questions first — prime the brain for what to catch.</li><li><strong>Learn → Do → Recall:</strong> Never learn a concept without immediately labbing it, then recalling it.</li><li><strong>The 10-minute rule for stuck states:</strong> If blocked &gt;10 min in a lab, read the solution, then redo from scratch.</li><li><strong>End-of-day export:</strong> Write 3 sentences on what you learned (generation effect) and 1 thing still fuzzy.</li></ul>

<code>create</code> fails if the object exists.</li><li><strong>How to see all resource types?</strong> <code>kubectl api-resources</code>.</li><li><strong>How to learn a field&#x27;s schema?</strong> <code>kubectl explain &lt;resource&gt;.spec...</code>.</li><li><strong>What is a namespace?</strong> A virtual cluster partition for scoping and isolating resources.</li><li><strong>Default namespaces?</strong> default, kube-system, kube-public, kube-node-lease.</li><li><strong>How to set a namespace for all commands?</strong> <code>kubectl config set-context --current --namespace=&lt;ns&gt;</code>.</li><li><strong>What is a kubeconfig context?</strong> A named bundle of cluster + user + namespace.</li><li><strong>How does the apiserver authenticate?</strong> Certificates, bearer tokens, or OIDC, then authorization (RBAC) and admission.</li><li><strong>What are admission controllers?</strong> Plugins that intercept API requests after authz to validate/mutate objects.</li><li><strong>Why is the API server stateless?</strong> All state lives in etcd; apiservers can scale horizontally.</li><li><strong>What talks directly to etcd?</strong> Only kube-apiserver.</li><li><strong>What is a control loop&#x27;s period?</strong> Continuous/event-driven watch, not a fixed poll.</li><li><strong>What is cloud-controller-manager?</strong> Integrates cluster with cloud provider APIs (load balancers, nodes, routes).</li><li><strong>What is a node&#x27;s status made of?</strong> Conditions (Ready, MemoryPressure...), capacity, allocatable, addresses.</li><li><strong><code>kubectl get -o wide</code>?</strong> Adds extra columns like node, IP.</li><li><strong><code>kubectl describe</code> vs <code>get -o yaml</code>?</strong> <code>describe</code> gives a human-readable summary incl. events; <code>-o yaml</code> gives raw object.</li><li><strong>How to see cluster events?</strong> <code>kubectl get events --sort-by=.lastTimestamp</code>.</li><li><strong>What is the CRI?</strong> Container Runtime Interface — gRPC contract between kubelet and runtime.</li><li><strong>CNI?</strong> Container Network Interface — plugin standard for pod networking.</li><li><strong>CSI?</strong> Container Storage Interface — plugin standard for storage.</li><li><strong>What ensures HA of the control plane?</strong> Multiple apiservers + a quorum (odd number) of etcd members.</li><li><strong>etcd quorum for 3 nodes?</strong> 2 (tolerates 1 failure).</li><li><strong>What is a manifest&#x27;s <code>metadata.labels</code> for?</strong> Key/value tags used by selectors to group objects.</li><li><strong>Annotations vs labels?</strong> Labels are for selection/queries; annotations hold non-identifying metadata.</li><li><strong>How does self-healing work?</strong> Controllers recreate/reschedule resources when actual drifts from desired.</li><li><strong>Is the scheduler responsible for running the pod?</strong> No — it only binds pod to node; kubelet runs it.</li><li><strong>What port does the apiserver typically serve?</strong> 6443 (HTTPS).</li><li><strong>How to check control-plane component health?</strong> <code>kubectl get componentstatuses</code> / check static pods in kube-system.</li><li><strong>What are static pods?</strong> Pods managed directly by kubelet from a manifest dir, not the apiserver.</li><li><strong>What runs the control plane in kubeadm clusters?</strong> Static pods on control-plane nodes.</li><li><strong>What is idempotency and why does it matter?</strong> Same operation yields same result; enables safe re-apply and reconciliation.</li><li><strong>What&#x27;s the smallest deployable unit?</strong> A Pod (not a container).</li><li><strong>One sentence: what is Kubernetes?</strong> A declarative, self-healing control system that keeps your containerized workloads matching the state you declared.</li></ol></div></details>

<details class="qa"><summary><span class="qa-badge">50 Q</span>50 Questions &amp; Answers<span class="chev">▾</span></summary><div class="qa-body"><ol><li><strong>What is a resource request?</strong> The minimum guaranteed CPU/memory used for scheduling decisions.</li><li><strong>What is a resource limit?</strong> The hard cap a container may use.</li><li><strong>What happens if a container CPU).</li><li><strong>How is memory measured?</strong> Bytes with suffixes (Mi, Gi).</li><li><strong>What is nodeSelector?</strong> Simplest node constraint via label matching.</li><li><strong>Node affinity vs nodeSelector?</strong> Affinity is more expressive (required/preferred, operators).</li><li><strong>requiredDuringScheduling...?</strong> Hard rule; pod won&#x27;t schedule if (topology).</li><li><strong>Pod anti-affinity?</strong> Keep pods apart (e.g., spread replicas across nodes).</li><li><strong>What is a taint?</strong> A node marking that repels pods lacking a matching toleration.</li><li><strong>What is a toleration?</strong> A pod&#x27;s permission to schedule onto a tainted node.</li><li><strong>Taint effects?</strong> NoSchedule, PreferNoSchedule, NoExecute.</li><li><strong>NoExecute effect?</strong> Evicts already-running pods without the toleration.</li><li><strong>Why taint control-plane nodes?</strong> To keep workloads off them by default.</li><li><strong>Topology spread constraints?</strong> Evenly distribute pods across zones/nodes (maxSkew).</li><li><strong>Scheduler phases?</strong> Filtering (feasible nodes) then Scoring (best node).</li><li><strong>What is preemption?</strong> Evicting lower-priority pods to fit a higher-priority pending pod.</li><li><strong>What is a PriorityClass?</strong> Assigns scheduling priority to pods.</li><li><strong>What is a PodDisruptionBudget?</strong> Limits voluntary disruptions (min available / max unavailable).</li><li><strong>Does a PDB protect against node crashes?</strong> No — only voluntary disruptions (drains, upgrades).</li><li><strong>What is metrics-server?</strong> Cluster-wide resource metrics source for HPA and <code>kubectl memory/custom/external).</li><li><strong>HPA formula?</strong> desiredReplicas = ceil(current × currentMetric / targetMetric).</li><li><strong>What is VPA?</strong> Vertical Pod Autoscaler — adjusts requests/limits (not replica count).</li><li><strong>Can HPA and VPA both target CPU?</strong> Not simultaneously on the same metric — they conflict.</li><li><strong>Cluster Autoscaler?</strong> Adds/removes nodes based on pending pods and utilization.</li><li><strong>What blocks node scale-down?</strong> Pods without controllers, PDBs, local storage, restrictive affinity.</li><li><strong>How system-reserved.</li><li><strong>What triggers node-pressure eviction?</strong> Memory/disk/PID pressure thresholds on kubelet.</li><li><strong>What is <code>kubectl cordon</code>?</strong> Marks a node unschedulable.</li><li><strong>What is <code>kubectl drain</code>?</strong> Evicts pods and cordons a node (for maintenance).</li><li><strong>What honors a PDB during drain?</strong> The eviction scale-down.</li><li><strong>Custom metrics for HPA source?</strong> Custom Metrics API (e.g., Prometheus Adapter).</li><li><strong>External metrics example?</strong> Queue length from a cloud service.</li><li><strong>Why can over-committing be fine?</strong> Requests reserve; actual usage may be lower, improving density.</li><li><strong>One-liner scaling stack?</strong> metrics-server → HPA (pods) + Cluster Autoscaler (nodes); VPA tunes per-pod resources.</li></ol></div></details>

<details class="qa"><summary><span class="qa-badge">50 Q</span>50 Questions &amp; Answers<span class="chev">▾</span></summary><div class="qa-body"><ol><li><strong>Monitoring vs observability?</strong> Monitoring watches known failure modes; observability lets you ask new questions about unknown states from telemetry.</li><li><strong>Three pillars?</strong> Metrics, logs, traces.</li><li><strong>When to use metrics?</strong> Cheap, aggregatable trends and alerting (is something wrong?).</li><li><strong>When to use traces?</strong> Following a request across services (where is it slow?).</li><li><strong>When to use logs?</strong> Detailed per-event context (what exactly happened?).</li><li><strong>Four golden signals?</strong> Latency, traffic, errors, saturation.</li><li><strong>RED method?</strong> Rate, Errors, Duration — for request-driven services.</li><li><strong>USE method?</strong> Utilization, Saturation, Errors — for resources.</li><li><strong>Prometheus data model?</strong> Time series identified by metric name + label set.</li><li><strong>Pull vs push?</strong> Prometheus pulls (scrapes) targets over HTTP; Pushgateway is for short-lived jobs.</li><li><strong>Counter?</strong> Monotonically increasing value (resets to 0 on restart).</li><li><strong>Gauge?</strong> A value that can go up and down (e.g., memory).</li><li><strong>Histogram?</strong> Buckets of observations + sum + count; enables quantile estimation.</li><li><strong>Summary?</strong> Client-side computed quantiles + sum + count.</li><li><strong>Histogram vs summary tradeoff?</strong> Histograms aggregate across instances; summaries don&#x27;t but are exact per-instance.</li><li><strong>What is exposition format?</strong> Plain-text <code>metric{labels} value</code> lines at <code>/metrics</code>.</li><li><strong>What is a scrape?</strong> Prometheus HTTP GET of a target&#x27;s metrics endpoint at an interval.</li><li><strong>What is service discovery?</strong> Auto-finding targets (Kubernetes SD, file SD, etc.).</li><li><strong>What is a label?</strong> A key/value dimension differentiating time series.</li><li><strong>What is cardinality?</strong> Number of unique time series; high cardinality kills performance.</li><li><strong>A cardinality anti-pattern?</strong> Putting user IDs, emails, or request IDs in labels.</li><li><strong>What is <code>rate()</code>?</strong> Per-second average increase of a counter over a range.</li><li><strong>Why use rate on counters, not raw values?</strong> Counters only increase; to reach root cause.</li><li><strong>Correlation vs causation?</strong> Co-occurrence isn&#x27;t cause; verify with a test/change.</li><li><strong>What is bisection in time?</strong> Narrowing to the deploy/config change that introduced the issue.</li><li><strong>What is bisection in space?</strong> Narrowing to the failing service/component.</li><li><strong>What is a cascading failure?</strong> One component&#x27;s failure overloading others, spreading collapse.</li><li><strong>What causes retry storms?</strong> Aggressive retries amplifying load during failures.</li><li><strong>Mitigation for retry storms?</strong> Backoff + jitter, circuit breakers, retry budgets.</li><li><strong>What is a thundering herd?</strong> Many clients hitting a resource simultaneously (e.g., cache expiry).</li><li><strong>Thundering herd fix?</strong> Jitter, request coalescing, staggered expiry.</li><li><strong>What is a circuit breaker?</strong> Stops calls to a failing dependency to allow recovery.</li><li><strong>What is a timeout&#x27;s role?</strong> Bounds waiting so failures don&#x27;t hang and cascade.</li><li><strong>What is saturation?</strong> How &quot;full&quot; a resource is (queues, CPU, connections).</li><li><strong>Latency triage first question?</strong> Is it all requests or a subset (endpoint/tenant/region)?</li><li><strong>How to find the slow service in a trace?</strong> Longest span on the critical path.</li><li><strong>How to confirm a hypothesis?</strong> Change one variable and observe the predicted effect.</li><li><strong>Why change one variable at a time?</strong> To attribute cause unambiguously.</li><li><strong>What is a red herring?</strong> A correlated-but-unrelated signal that misleads.</li><li><strong>What is tail latency?</strong> High-percentile (p99) latency affecting a minority of requests.</li><li><strong>Why does p99 matter more than avg?</strong> Averages hide user-facing worst cases.</li><li><strong>What is a hotspot?</strong> A disproportionately loaded shard/instance/key.</li><li><strong>How to detect a memory leak?</strong> Steadily rising memory + eventual OOMKills.</li><li><strong>How to detect CPU throttling in K8s?</strong> Latency up with <code>container_cpu_cfs_throttled</code> increasing.</li><li><strong>How to spot a noisy neighbor?</strong> Contention correlating with another workload on the same node.</li><li><strong>What is blast radius?</strong> The scope of impact of a failure/change.</li><li><strong>How to reduce blast radius?</strong> Canaries, cell/shard isolation, feature flags, gradual rollout.</li><li><strong>What is a canary?</strong> Releasing to a small subset first to catch regressions.</li><li><strong>What is a rollback-first mindset?</strong> Restore service by reverting, then investigate.</li><li><strong>Mitigate vs fix?</strong> Mitigate stops user pain now; fix addresses root cause after.</li><li><strong>What is a runbook?</strong> Step-by-step guidance for a known alert/scenario.</li><li><strong>How to debug intermittent issues?</strong> Increase sampling/logging, add tracing, reproduce under load.</li><li><strong>What is chaos testing&#x27;s debugging value?</strong> Reveals failure modes before they happen in prod.</li><li><strong>How to debug DNS in K8s?</strong> Exec pod → nslookup; check CoreDNS + NetworkPolicies.</li><li><strong>How to debug a stuck rollout?</strong> <code>rollout status</code>, events, readiness probes, image/pull errors.</li><li><strong>How to debug intermittent 5xx?</strong> Trace error requests, inspect logs by trace_id, check dependency saturation.</li><li><strong>What is a dependency graph?</strong> Map of service calls; essential for reasoning about cascades.</li><li><strong>How to detect a bad deploy fast?</strong> Deploy annotations on dashboards + SLO burn alerts.</li><li><strong>What is a &quot;known-good&quot; comparison?</strong> Diffing behavior/config against a healthy baseline/region.</li><li><strong>How to avoid confirmation bias?</strong> Try to disprove your hypothesis, not just confirm it.</li><li><strong>What to capture during an incident?</strong> Timeline, actions, signals, decisions (for the postmortem).</li><li><strong>What is time-to-detect (TTD)?</strong> Time from failure start to detection.</li><li><strong>What is time-to-mitigate (TTM)?</strong> Time from detection to restored service.</li><li><strong>How does good observability shrink MTTR?</strong> Faster localization via correlated pillars.</li><li><strong>What&#x27;s the debugging anti-pattern?</strong> Randomly changing things without hypotheses (&quot;shotgun debugging&quot;).</li><li><strong>One-liner method?</strong> Start from the symptom, use metrics→traces→logs to localize, test one hypothesis at a time, mitigate fast, then fix and document.</li></ol></div></details>

nuance?</strong> Exclude events you shouldn&#x27;t be judged on (e.g., 4xx client errors, sometimes).</li><li><strong>Availability SLI formula?</strong> successful requests / valid requests.</li><li><strong>Latency SLI formulation?</strong> % of requests faster than threshold (e.g., 95% &lt;
